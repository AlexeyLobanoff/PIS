# ETL-приложение: парсинг текстовых данных в MongoDB

Десктопное приложение для парсинга специфических текстовых данных (разделитель `;`) и загрузки в MongoDB с отчётами.

## Стек

- **GUI:** tkinter / customtkinter (при установленном пакете)
- **БД:** pymongo (подключение по URI)
- **Отчёты:** pandas, openpyxl, xlsxwriter

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

## Подключение к MongoDB

В поле URI укажите строку подключения. Если сервер требует аутентификацию, обязательно укажите логин и пароль в URI, например:

```
mongodb://пользователь:пароль@localhost:27017/
```

Без учётных данных при включённой аутентификации на сервере операции записи завершатся ошибкой «command insert requires authentication».

## Формат входного файла

Текстовый файл, разделитель полей — `;`.

- **Поля 1–4 (фиксированные):** лицевой счёт (int), ФИО, адрес (внутри — запятая, минимум 3 сегмента: населённый пункт, улица, дом), период (int).
- **Поле 5 (вариативное):**
  - **Сценарий А:** одно поле после периода — сумма (float).
  - **Сценарий Б:** группы по 3 поля (сумма, название прибора, показание). Количество групп от 1 до N.

Строка невалидна, если после периода полей не 1 и не кратно 3.

## Структура проекта

| Файл        | Назначение                                      |
|------------|--------------------------------------------------|
| `main.py`  | Точка входа, GUI (выбор файла, логи, прогресс, статистика) |
| `parser.py`| Класс `DataProcessor`: чтение файла, валидация, разбор строк |
| `database.py` | Класс `MongoManager`: подключение к MongoDB, `insert_one` |
| `reports.py`  | `errors_report.txt`, экспорт в Excel с круговой диаграммой |

## Функциональность

1. **Парсинг и валидация:** типы (счёт, период — int; суммы, показания — float), адрес ≥3 сегмента, замена запятой на точку в числах.
2. **MongoDB:** документ = фиксированные поля + массив `entries` для вариативной части; вставка по одной записи.
3. **Интерфейс:** выбор файла, лог в реальном времени, прогресс-бар, статистика (обработано / успешно % / ошибки %).
4. **Отчёты:** `errors_report.txt` с причиной по каждой ошибочной строке; экспорт в Excel с круговой диаграммой по статистике обработки.

Обработка исключений: `ConnectionFailure`, ошибки кодировки (utf-8, cp1251); ошибки логируются и отображаются в GUI без падения приложения.
